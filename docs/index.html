2 Haskell学習にあたって

2.1 Haskellの概要

Haskellはかなり現代的な思想を取り入れた言語であるが、その歴史は非常に古く、登場時期は1990年頃である。 Haskellは他のプログラミング言語でできることを網羅した上、純粋関数型プログラミングや強い静的型付け、遅延評価などユニークな特徴を多く持つ。 C言語やJava、Pythonのような人気プログラミング言語とは決して言えないが、コンピューターサイエンスなどの研究対象になることが多い。 また、Haskellの高い信頼性によって、金融業界で採用された例もある。

2.2 Haskellを学ぶ理由

Haskellは決して万人受けするような言語ではなく、習得したところで仕事がたくさんあるとは言えない。 学習コストも高く、他の言語で得た知識が通用しない場面もある。 しかし、筆者はHaskellのことを世界で最も素晴らしいプログラミング言語の1つであると考えている。 Haskellの思想は、プログラムの挙動を隅から隅まで把握しようとするものであったり、間違ったプログラムをコンピュータに指摘させようとするものである。 この思想をプログラマに押し付けてくる点も、Haskellの難しさであるが、慣れてしまえば他の言語もHaskell風に書きたくなるほど強力で実用的なものである。 Haskellで記述したプログラムのバグのなさに怖くなる経験をするかもしれない。 Haskellの最大の利点はプログラムの信頼性・保守性であるから、信頼性が求められるような場面に際して、Haskellを学ぶことには十分意味があるといえるだろう。 そして、プログラミング初学者においても、Haskell学習の過程で得たコンピューターサイエンスの知識・数学的思考は、プログラミングの考え方の強力な根幹となる。 したがって、ハードルが高いと思うかもしれないが、初学者にもHaskellを学ぶことをお勧めしたい。

3 環境構築

このテキストでは、Windows 10以上を搭載したパソコンを対象としている。 手順をLinux用・macOS用に読み替えられる場合を除いて、Windowsを使用することを推奨する。(なぜなら、筆者がパソコンに全く詳しくないからである。)

3.1 Visual Studio Codeの導入

まず、MicrosoftのサイトからVisual Studio Codeのインストーラを入手し、インストールする必要がある。 特に注意すべき点はない。

3.2 GHCupの導入

次に、Haskellコンパイラをはじめ様々なツールをインストールしてくれる、GHCupを導入する。 GHCupのサイトにアクセスし、Set-ExecutionPolicyから始まるコマンドをコピーしよう。 そして、コピーしたコマンドを管理者権限ではないPowerShellに貼り付け、実行しよう。(コマンドプロンプトでは不可能であることに注意。) 実行すると、GHCupインストーラからいくつかの質問が来るはずである。 下に、筆者がおすすめする回答を記しておく。
  • Press enter to accept the default [C:\]: → (何も入力せずEnter)
  • Press enter to accept the default [C:\\cabal]: → (何も入力せずEnter)
  • Do you want to install the haskell-language-server (HLS) for development purposes as well? → Y
  • Do you want to install stack as well? → Y
  • Do you want to create convenience desktop shortcuts (e.g. for uninstallation and msys2 shell)? → N
  • Do you want GHCup to install a default MSys2 toolchain (recommended)? → Y
ここまでの質問に答えた後、しばらくの間Linuxのような画面が流れるはずである。MinGW x64というウインドウが立ち上がってきたら、PowerShellを閉じてもよい。 ここで、MinGW x64を閉じてはいけない!! MinGW x64のウインドウに、白文字でPress any key to exitと出たら、インストールは成功である。何らかのキーを押して終了しよう。

3.3 拡張機能の導入

次に、Visual Studio CodeにHaskell用の拡張機能を導入する。 既に開かれているVisual Studio Codeがある場合、それらを一度閉じて、新しくVisual Studio Codeを起動しよう。 Visual Studio Codeを起動したら、画面左の中央あたりにある4つの四角形で構成されたアイコンをクリックしよう。 (マウスカーソルを重ねると、Extensions (Ctrl+Shift+X)と表示されるはずである。) そのアイコンをクリックすると、Search Extensions in Marketplaceと書かれているであろう検索欄が現れるはずである。 そこにhaskellと入力し、一番上に出てくるHaskellという項目を選択する。 (説明文がHaskell language support powered by the Haskell Language Serverとなっていることを確認しよう。) Installボタンをクリックし、DisableボタンとUninstallボタンが現れれば、拡張機能の導入は完了である。

3.4 Visual Studio Codeの設定

次に、Visual Studio Codeの設定を行う。 Visual Studio Codeの画面左下にある歯車アイコンをクリックし、Settingsを選択する。 Search settingsと書かれている検索欄に、manage hlsと入力し、Haskell: Manage HLSという項目を探そう。 PATHと書かれている場所をクリックし、表示されたメニューからGHCupを探してクリックしよう。 これで、Haskellに入門する準備が整った。

3.5 サンプルプログラムの実行

好きな場所にHaskellのソースファイルを作成しよう。 ファイル名は自由であるが、拡張子は.hsでなくてはならない。 作成した空のソースファイルをVisual Studio Codeで開く。 この時、Visual Studio CodeからNeed to download hls-???, continue?と聞かれた場合、Yesを選択しよう。 そして、下記のサンプルプログラムを入力して、保存しよう。 Visual Studio Codeのメニューから、File → Auto Saveを選択し、オートセーブを有効にしておくと便利である。 main :: IO () main = putStrLn "Hello World." 次に、Visual Studio CodeのメニューからTerminal → New Terminalを選択する。 画面下に表示されたPowerShellで、ソースファイルがある場所まで移動し、stack runhaskell ファイル名.hsを実行しよう。 最後にHello World.と出力されたなら、サンプルプログラム実行は成功である。 初回はかなり時間がかかるが、2回目以降はすぐにプログラムが実行されるだろう。
